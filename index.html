Monte-Carlo Ray Tracer

Source Code

    C++
    Global Illumination
    Path Tracing
    Rendering Equation

This is a Monte-Carlo Path Tracer made in the course Advanced Global Rendering and Illumination at Linköping University by me and another student. This implementation solves the rendering equation using Monte-Carlo integration by supersampling each pixel. This is still an ongoing project where some of the current implemented features are: Explicit light sampling, Ray-triangle intersection with Möller Trumbore, Area and point lights, Ray-sphere intersection, Loader for .obj, Multi-threading and Lambertian, Specular and Transparent BRDFs. The resulting image is generated using 10,000 samples with a ray depth of 7.

Done!
Maya Hair Plug-in

Source Code

    C++
    Maya API
    Physics
    Simulation
    Numerical Solver

In the course SFX - Tricks of the Trade, me and two other students created a plug-in to Maya using their C++ API with the purpose of simulating and rendering realistic hair. The implementation is based on the paper A Mass Spring Model for Hair Simulation by Selle et al. The strands were created as Nurbs Curves in Maya, uniformly attached to a sphere and connected to the plug-in using a MEL script. In the plug-in we introduced different types of springs such as altitude and torsion springs to mimic the real behaviour of hair together with a forward Euler time step connected to the time slider in Maya. The rendering of the hair was done using the Arnold ray tracer introduced in Maya 2017 and the final result is using 30,000 strands.

Done!
Procedural Water Shading in Real-time

Source Code - Report - Demo

    GLSL
    WebGL
    JavaScript
    Water Simulation
    Real-time

This is an individual project in the course Procedural Methods for Images at Linköping University where I created procedurally generated water and terrain without using any textures, completely in GLSL. The ripples of the water and the noise of the terrain are generated with the help of Simplex Noise and the terrain and bottom are created with a 2D Gaussian function. The reflection and refraction are generated using multi pass rendering with clipping planes together with projective texturing. A simplified Fresnel effect was implemented to generate more reflection when looking parallel to the surface plane. Phong lightning model are applied to perturbed normals by recalculating them using the gradient of the noise function in object space. I also implemented a GUI to be able to tweak some of the parameters in real time, see the report for all details. A live demo can be found here.

Done!
SPH Fluid Simulation

Source Code - Report

    C++
    OpenGL
    Physics
    Simulation
    Numerical Solver

In the course Modeling and Simulation at Linköping University, me and three other students implemented a real-time fluid simulation using Navier-Stokes together with Smoothed Particle Hydrodynamics. The fluid was firstly created in 2D, but was also extended to work in 3D using OpenCL libraries. The implementation was done in C++ and OpenGL using a grid system for optimized neighbour lookups. The rendering was done in 2D using the marching squares algorithm. I also ported parts of the project to JavaScript for fun, this demo can be found here.

AI using Genetic Algorithms

Source Code - Report

    Java
    Genetic Algorithms
    AI

In this project me and three other students built a Tetris AI using an utility function based on a linear weighted sum of heuristics. The parameters for the heuristics were obtained by running a genetic algorithm and the AI improved its skill by replacing the current best values with new ones by running multiple evolutions on different threads.

A population was created with randomly generated unit vectors with a dimension of the number of heuristics which were: Number of Holes, Landing Height, Row Transitions, Column Transitions, Well Sums and Completed Lines. Each evolution the weakest 30% were replaced and each newly produced offspring had a 10% chance to mutate. The evolution was run until the fitness function gave the expected result. The AI managed to clear an average of ~2 million lines per game.

Done!

Source Code - Report

    OCaml
    Functional Programming

This is a compiler written in OCaml for the language mOOL made in the module Compiler Design at the National University of Singapore. The phases implemented were: Lexical, Syntax and Semantic Analysis, Intermediate Code Generation, Code Optimization and Code Generation. The phases Lexical, Syntax and Semantic Analysis were individual and Intermediate Code Generation, Code Optimization and Code Generation were implemented together with two other students.
